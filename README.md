# dota-predictor

Предсказание вероятности победы в профессиональных матчах Dota 2.

## Быстрый старт

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -e .
python -m dota_predictor.pipeline --pages 20
```

Пайплайн скачивает последние про-матчи с [OpenDota API](https://docs.opendota.com/)
(кэширует в `data/raw/`), считает Elo-рейтинги и форму команд, обучает
логистическую регрессию и сравнивает её с наивными baseline'ами по
accuracy / log loss / Brier score на временном сплите (train — прошлое,
test — будущее).

## Структура

```
src/dota_predictor/
├── ingest/      # OpenDota: REST + bulk-выгрузки через /explorer SQL; кэфы из CSV
├── features/    # Elo, форма, драфты, мета патча/турнира, баны, составы — без утечек
├── models/      # baseline, CatBoost, калибровка, in-game модель по золоту
├── llm/         # агент-аналитик на Anthropic tool runner
├── api/         # FastAPI: /predict, /preview, /teams
├── live.py      # live-трекер идущих матчей (OpenDota /live)
└── pipeline.py  # точка входа обучения
```

## Roadmap

- [x] v0: Elo + форма, логистическая регрессия
- [x] v1: драфт-фичи (винрейты героев, bag-of-heroes), CatBoost
      (драфты тянутся пачками через `/explorer` SQL endpoint)
- [x] v1.1: фильтр по тиру лиги (`--tiers premium,professional` по умолчанию);
      фичи считаются по всем матчам, обучение/оценка — только по выбранным тирам
- [x] v1.2: глубокая история через `/explorer` по таблице `matches`
      (`--history-days 730` по умолчанию, ~54 тыс. матчей); затухание
      винрейтов героев с half-life 90 дней под смену патчей
- [x] v2: калибровка (Platt/isotonic, ECE, reliability-таблица); сравнение
      с кэфами через `--odds-csv` (формат `match_id,odds_radiant,odds_dire`,
      вилка снимается нормализацией, есть симуляция флэт-ставок)
- [x] v3: LLM-агент превью матчей — Claude (Anthropic SDK, tool runner) сам
      собирает контекст инструментами (обзор команд, личные встречи, прогноз
      модели) и пишет аналитическое превью:
      ```powershell
      $env:ANTHROPIC_API_KEY = "sk-ant-..."
      python -m dota_predictor.llm.preview "Team Spirit" "Team Falcons"
      ```
- [x] v4: FastAPI-сервис:
      ```powershell
      .venv\Scripts\python -m uvicorn dota_predictor.api.app:app --port 8123
      ```
      Эндпоинты: `GET /health`, `GET /teams?query=...` (поиск команды),
      `GET /predict?team_a=...&team_b=...` (вероятность + форма + личные
      встречи), `GET /preview?...` (LLM-превью; нужен ANTHROPIC_API_KEY,
      иначе 503). Интерактивная документация — `/docs`.
- [x] v5: мета патча и контекст турнира — винрейты/популярность героев
      внутри текущего патча (сброс на границе патча), форма команды на
      текущем турнире до матча, винрейты героев на турнире. Форма на
      турнире — вторая по важности фича модели после Elo.
- [x] v5.1: баны из `picks_bans` — сила и «метовость» забаненных героев,
      целевые баны (насколько соперник выцеливает сигнатурных героев
      команды в текущем патче).
- [x] v6: составы игроков из `player_matches` — личный винрейт пятёрки,
      сыгранность (матчи именно этой пятёркой), стабильность состава
      (детектор стендинов). Личный винрейт игроков обошёл Elo команды
      и стал важнейшей фичей модели; лучший log loss 0.6465 (с Platt).
- [x] v7: live-трекер (`python -m dota_predictor.live [--watch 60]
      [--odds 1.45,2.75]`) — идущие про-матчи из OpenDota `/live`:
      предматчевая вероятность карты + текущее преимущество по золоту;
      сравнение с букмекерской линией через `--odds`.
- [x] v7.1: обученная in-game модель (`python -m dota_predictor.models.ingame`) —
      CatBoost на 163 тыс. поминутных срезов графиков золота 24.7 тыс.
      матчей; log loss 0.49 против 0.55 у ручной эвристики и 0.66 у
      предматчевого прогноза; accuracy 75% по ходу карты. Вывод из
      модели: золото в начале игры весит больше, чем в конце (5k на
      10-й минуте = 77% победы, на 40-й = 68%).

## Кейс-стади: гранд-финал Esports World Cup 2026

Живая проверка всех слоёв системы на реальном турнире (Париж, 19 июля
2026): **BoomBoys vs PVISION**, bo5. Прогнозы зафиксированы до и во время
серии, разбор написан после.

**До серии.** Elo 1983 (ранг 3) против 1919 (ранг 4), форма 10/10 против
8/10, личные встречи 2:2. Модель: BoomBoys берут карту с 56.1%, титул —
с 61.3%. Матч равных: ни одной из команд не дано больше 61%.

**По ходу серии** сценарная таблица bo5 обновляла титульные шансы
BoomBoys: 1:0 → **77.3%**, 1:1 → 59.1%, 1:2 → 31.5%. Итог — **PVISION
3:1**: сбылся сценарий, которому модель давала 38.7%.

**Прогнозы по картам** (вероятность, выданная фактическому победителю):

| Карта | Прогноз победителю | Факт |
|---|---|---|
| 1 | BoomBoys 57% | ✓ |
| 2 | PVISION 42% | ✗ |
| 3 | PVISION 45% | ✗ |
| 4 | PVISION 50% | — монетка |

**In-game модель** (v7.1, обучена на графиках золота) — вероятность
итогового победителя карты по минутам:

| Карта | 10 мин | 20 мин | 30 мин | 40 мин |
|---|---|---|---|---|
| 1 (BoomBoys) | 80% | 76% | 76% | 70% |
| 2 (PVISION)  | 34% | 43% | 46% | 86% |
| 3 (PVISION)  | 51% | 74% | 80% | 96% |

На всех картах к 40-й минуте модель уверенно указывала настоящего
победителя. Карта 2 — самый показательный эпизод: PVISION реально
проигрывали первые 20 минут (модель честно держала их ниже 50%), затем
перевернули игру — и модель отследила камбэк до 86%.

**Выводы.** (1) Предматчевый слой корректно классифицировал финал как
матч равных; проигрыш фаворита с P=61% — штатное событие для
калиброванной модели, а не ошибка (в тестах события из бина 60–70%
сбываются в ~64% случаев). (2) Слабое звено — выбор стороны фаворита:
без драфтов и инсайдов Elo+форма на равных командах почти не разделяют.
(3) In-game слой — самый сильный: обученная на 163 тыс. срезов модель
сопровождала все карты корректно, включая камбэк. (4) Идеальная форма
10/10 у проигравшего фаворита — иллюстрация отрицательного коэффициента
формы из v0: горячая серия против сильного соперника склонна к регрессии
к среднему.
